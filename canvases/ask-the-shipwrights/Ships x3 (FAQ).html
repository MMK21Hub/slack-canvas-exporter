What counts as a ship?

Any project that you’ve worked on yourself unless:

  • it was for school
  • it was paid
  • it’s closed source
  • it’s a 1:1 replica of a tutorial - remix it and make it your own! 

How do I ship it?

There are three essential parts to a shipped project:

  • A repository 
  • A demo
  • An image

Let's go through them one by one 

Repository

Your repository needs two things. The first is all of your amazing code! Everything we share at Hack Club is open source (even Summer of Making). In most cases your code will be on GitHub, but you're welcome to use any Git host.

The second thing you repository needs is a good README.md file. So, what is a README.md file? This is the first thing that people are when they load  repository. It should have all the information that anyone needs to know about your project in it. There's no strict list for what your readme should have, but aim to have the following. 

  • an image of your project
  • a description of your project
  • instructions on how to try your project out 
  • inspiration? Why'd you make your project?
  • technologies you've used - you don't need to list everything but if you're using a package for some of the main features, you should mention it

Remember that this is the place where you can talk about your project and say whatever you want!

Demo

This really depends what your project is! Different projects get shipped differently, but you want other people to be able to experience your project as easily as possible. Here are some examples of what is and isn’t okay.

  • Website
    • :white_check_mark: Deploy it on #C056WDR3MQR, GitHub Pages, Vercel, etc so we can visit a URL to see it
    • :red-x: Don't send the GitHub repo with instructions to download it and run a development server locally
  • Game
    • :white_check_mark: Build a release and share it on itch.io, GitHub Releases or similar for us to see it. Bonus points if you can deploy it to run as a website!
    • :red-x: Don't make us run and build it locally
  • Game Mod
    • :white_check_mark: Publish it on the appropriate website for the game you've modded! Common ones include Modrinth, Hangar, Curseforge, Nexus
    • :red-x: Don't make us build your mod locally! We want to test it out
  • Command line tool or package
    • :white_check_mark: Publish it on the package index for your language. PyPi for Python, NPM for JavaScript/TypeScript, crates.io for Rust, etc
    • :red-x: Don't make us download your project locally, install dependencies and run a specific file

Of course, your project might not be any of those! If you're not sure what would be best for your project, ask in #C090JKDJYN8 and we'll help you figure it out!

Image 

Show your project in action! When your project comes up in voting, the image is going to be the first thing they see, so you want to choose somehing that really shows what you've made.

What next?

Once you've shipped your project, the shipwrights will try out your ship! The shipwrights are a group of hack clubbers who want to try out your projects and make sure that they're shipped and work properly. If something isn't quite working then they'll let you know so you can fix it :)
