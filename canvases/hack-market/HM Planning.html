HM Planning

​

  • A real database (not airtable) to store data in
  • A command to make a new listing
  • A command to list all listings
  • A command to search listings
  • Once a day it automatically sends all listings from the previous day in a thread
  • Public API to view listings and currency
  • Private API to edit listings and currency using API keys

​

Idea for CLI to obtain API keys:

  • If someone wants to contribute, they'll get a magical keyword DMed, like 32 characters so it isn't easily de-hashable
  • There will be a Python CLI where people can, upon entering the magic keyword, create an account
  • With this account you can generate API keys with name and expiration date, all through the CLI
  • Account data will be stored in a seperate Supabase database, as will the API keys associated with the accounts
